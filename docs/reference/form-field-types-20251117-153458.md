## üìã FIELD PROPERTIES & CONFIGURATION

All field types support the following configurable properties:

### Basic Properties
- `fieldId`: Unique field identifier
- `fieldType`: Field type (from above list)
- `fieldName`: Form field name
- `label`: Field label
- `placeholder`: Placeholder text
- `helpText`: Help text
- `tooltip`: Tooltip text
- `required`: Required flag
- `disabled`: Disabled state
- `readonly`: Readonly state
- `hidden`: Hidden state
- `defaultValue`: Default value

### Validation Properties
- `validationRules`: Array of validation rules
- `minLength`: Minimum length
- `maxLength`: Maximum length
- `min`: Minimum value (for numbers)
- `max`: Maximum value (for numbers)
- `pattern`: Regex pattern
- `customValidator`: Custom validation function

### Styling Properties
- `className`: CSS class names
- `style`: Inline styles
- `containerClassName`: Container CSS classes
- `containerStyle`: Container inline styles
- `width`: Field width
- `height`: Field height
- `responsive`: Responsive behavior

### Behavior Properties
- `conditionalDisplay`: Conditional display rules
- `conditionalEnable`: Conditional enable rules
- `conditionalRequired`: Conditional required rules
- `onChange`: Change event handler
- `onBlur`: Blur event handler
- `onFocus`: Focus event handler
- `autocomplete`: Autocomplete behavior
- `autofocus`: Autofocus flag

### Options Properties (for Selects)
- `options`: Array of options
- `optionGroups`: Grouped options
- `allowOther`: Allow "Other" option
- `multiple`: Multiple selection
- `searchable`: Searchable dropdown
- `asyncLoad`: Async option loading

### File Upload Properties
- `accept`: Accepted file types
- `maxSize`: Maximum file size
- `maxFiles`: Maximum number of files
- `multiple`: Multiple file upload
- `dragDrop`: Drag-drop enabled
- `preview`: Preview enabled

### Date/Time Properties
- `format`: Date/time format
- `minDate`: Minimum date
- `maxDate`: Maximum date
- `disabledDates`: Disabled dates
- `timeFormat`: Time format
- `timezone`: Timezone

## üîÑ CONDITIONAL LOGIC & DEPENDENCIES

All field types support conditional display/enable/required logic:

### Conditional Rules
- **When**: Condition expression
- **Show**: Show field when condition true
- **Hide**: Hide field when condition false
- **Enable**: Enable field when condition true
- **Disable**: Disable field when condition false
- **Required**: Make required when condition true
- **Optional**: Make optional when condition false

### Dependency Types
- **Field Dependency**: Depends on another field value
- **Form State Dependency**: Depends on form state
- **Data Dependency**: Depends on external data
- **User Role Dependency**: Depends on user role
- **Step Dependency**: Depends on current step (multi-step)
