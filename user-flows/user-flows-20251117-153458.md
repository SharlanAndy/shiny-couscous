## üìã FORM SUBMISSION FLOWS

### Flow 1: Form Discovery & Selection

**Trigger**: User wants to submit a Labuan FSA application form

**Steps**:
1. User visits Labuan FSA website ‚Üí Clicks "E-Submission System"
2. **Landing Page** ‚Üí User sees system overview and available forms
3. **Form List Page** ‚Üí User browses all available forms
4. **Form Detail Page** ‚Üí User views form description, requirements, estimated time
5. **Decision Point**:
   - If user is logged in ‚Üí Proceed to Flow 2 (Form Filling)
   - If user is not logged in ‚Üí Proceed to Flow 1A (Registration/Login)

**Success Criteria**: User successfully selects a form and is ready to fill it out

**Error Paths**:
- **Error**: Form list fails to load
  - **Recovery**: Display error message, retry button, contact support link
- **Error**: User selects wrong form
  - **Recovery**: Back button, form comparison tooltip

**Pages Involved**: Landing Page, Form List Page, Form Detail Page

---

### Flow 2: User Registration

**Trigger**: User wants to create an account to submit forms

**Steps**:
1. **Registration Page** ‚Üí User clicks "Create Account"
2. **Registration Form** ‚Üí User fills out:
   - Full Name (required)
   - Email Address (required, validated)
   - Password (required, strength indicator)
   - Confirm Password (required, match validation)
   - Phone Number (optional)
   - Terms & Conditions checkbox (required)
3. **Email Verification** ‚Üí User receives verification email
4. **Verification Page** ‚Üí User enters verification code OR clicks email link
5. **Account Created** ‚Üí User redirected to dashboard

**Success Criteria**: User successfully creates account and verifies email

**Error Paths**:
- **Error**: Email already exists
  - **Recovery**: Display error, offer login link
- **Error**: Password too weak
  - **Recovery**: Show strength requirements, highlight weak areas
- **Error**: Verification code expired
  - **Recovery**: Resend verification email option

**Pages Involved**: Registration Page, Email Verification Page, Verification Code Page

---

### Flow 3: User Login

**Trigger**: User wants to access their account

**Steps**:
1. **Login Page** ‚Üí User clicks "Sign In"
2. **Login Form** ‚Üí User enters:
   - Email Address (required)
   - Password (required)
   - Remember Me checkbox (optional)
3. **Authentication** ‚Üí System validates credentials
4. **Dashboard** ‚Üí User redirected to personal dashboard

**Success Criteria**: User successfully logs in and accesses dashboard

**Error Paths**:
- **Error**: Invalid email or password
  - **Recovery**: Display error, offer "Forgot Password" link
- **Error**: Account locked (too many failed attempts)
  - **Recovery**: Display lockout message, contact support link
- **Error**: Account not verified
  - **Recovery**: Display verification prompt, resend email option

**Pages Involved**: Login Page, Dashboard

---

### Flow 4: Form Filling (Single-Step Form)

**Trigger**: User selects a single-step form to fill out

**Steps**:
1. **Form Page** ‚Üí User sees form with all fields rendered from API
2. **Field Interaction** ‚Üí User fills out fields:
   - Text inputs (real-time validation)
   - Dropdowns (searchable, with "Other" option)
   - File uploads (drag-drop, preview)
   - Date pickers (calendar widget)
3. **Validation Feedback** ‚Üí Real-time validation errors displayed
4. **Help & Guidance** ‚Üí User clicks help icons for field explanations
5. **Draft Saving** ‚Üí Auto-save or manual "Save Draft" button
6. **Form Review** ‚Üí User reviews completed form
7. **Submission** ‚Üí User clicks "Submit" button
8. **Confirmation** ‚Üí Success message with submission ID

**Success Criteria**: User successfully completes and submits form

**Error Paths**:
- **Error**: Validation errors on submission
  - **Recovery**: Scroll to first error, highlight invalid fields
- **Error**: Network error during submission
  - **Recovery**: Retry button, auto-retry with exponential backoff
- **Error**: File upload fails
  - **Recovery**: Retry upload, show file requirements

**Pages Involved**: Form Page, Form Review Page, Submission Confirmation Page

---

### Flow 5: Form Filling (Multi-Step Form)

**Trigger**: User selects a multi-step form to fill out

**Steps**:
1. **Form Page - Step 1** ‚Üí User sees first step with progress indicator
2. **Step Completion** ‚Üí User fills out Step 1 fields, clicks "Next"
3. **Step Validation** ‚Üí System validates Step 1 before proceeding
4. **Form Page - Step 2** ‚Üí User sees second step, progress updated
5. **Step Navigation** ‚Üí User can go back/forward between steps
6. **Step N Completion** ‚Üí User completes all steps
7. **Form Review** ‚Üí User reviews all steps in summary view
8. **Final Submission** ‚Üí User submits complete form
9. **Confirmation** ‚Üí Success message with submission ID

**Success Criteria**: User successfully completes multi-step form and submits

**Error Paths**:
- **Error**: Step validation fails
  - **Recovery**: Stay on current step, highlight errors
- **Error**: User loses progress
  - **Recovery**: Auto-save drafts, resume from last step
- **Error**: Step navigation breaks
  - **Recovery**: Save progress, reload form state

**Pages Involved**: Multi-Step Form Page, Form Review Page, Submission Confirmation Page

---

### Flow 6: File Upload

**Trigger**: User needs to upload required documents

**Steps**:
1. **File Upload Field** ‚Üí User sees upload area in form
2. **File Selection** ‚Üí User clicks "Choose File" OR drags file to drop zone
3. **File Validation** ‚Üí System validates:
   - File type (PDF, JPG, PNG, DOC, DOCX)
   - File size (max 10MB)
   - File name
4. **Upload Progress** ‚Üí Progress bar shows upload status
5. **Upload Preview** ‚Üí Uploaded file preview displayed
6. **File Actions** ‚Üí User can:
   - View preview
   - Remove file
   - Replace file
7. **Upload Complete** ‚Üí File attached to form submission

**Success Criteria**: User successfully uploads required document(s)

**Error Paths**:
- **Error**: File type not supported
  - **Recovery**: Display accepted formats, allow re-selection
- **Error**: File too large
  - **Recovery**: Display size limit, compress suggestion
- **Error**: Upload fails
  - **Recovery**: Retry button, show error details

**Pages Involved**: Form Page (within form context)

---

### Flow 7: Submission Status Tracking

**Trigger**: User wants to check status of submitted form

**Steps**:
1. **Dashboard** ‚Üí User clicks "My Submissions"
2. **Submissions List** ‚Üí User sees all submissions with status badges:
   - Draft (editable)
   - Submitted (pending review)
   - Under Review (in progress)
   - Approved (completed)
   - Rejected (requires action)
3. **Submission Detail** ‚Üí User clicks submission to view details
4. **Status Timeline** ‚Üí User sees submission history:
   - Submitted: Date/Time
   - Under Review: Date/Time, Reviewer name
   - Decision: Approved/Rejected, Date/Time, Notes
5. **Actions** ‚Üí User can:
   - View submission details
   - Download submission PDF
   - Resubmit (if rejected)

**Success Criteria**: User successfully tracks submission status

**Error Paths**:
- **Error**: Submissions list fails to load
  - **Recovery**: Retry button, contact support
- **Error**: Submission detail not found
  - **Recovery**: Redirect to submissions list, error message

**Pages Involved**: Dashboard, Submissions List Page, Submission Detail Page

### Flow 8: Admin Login & Dashboard

**Trigger**: Administrator wants to access admin dashboard

**Steps**:
1. **Admin Login Page** ‚Üí Admin enters credentials
2. **Authentication** ‚Üí System validates admin credentials
3. **Admin Dashboard** ‚Üí Admin sees:
   - Statistics: Total submissions, pending, approved, rejected
   - Recent activity: Latest submissions and reviews
   - Quick actions: Review pending, manage forms
4. **Navigation** ‚Üí Admin accesses different sections:
   - Submissions Management
   - Form Schema Management
   - User Management
   - Audit Logs
   - Reports & Analytics

**Success Criteria**: Admin successfully accesses dashboard

**Error Paths**:
- **Error**: Invalid credentials
  - **Recovery**: Error message, "Forgot Password" link
- **Error**: Account locked
  - **Recovery**: Lockout message, contact system admin

**Pages Involved**: Admin Login Page, Admin Dashboard

---

### Flow 9: Review Submission

**Trigger**: Admin wants to review a pending submission

**Steps**:
1. **Admin Dashboard** ‚Üí Admin clicks "Review Pending Submissions"
2. **Submissions List** ‚Üí Admin sees filtered list:
   - Filter by: Form type, Status, Date range
   - Sort by: Date, Priority, Form type
   - Search: By submission ID, user name, form name
3. **Submission Detail** ‚Üí Admin clicks submission to review:
   - View form data (all fields)
   - Download uploaded documents
   - View submission history
4. **Review Actions** ‚Üí Admin can:
   - Approve submission
   - Reject submission (with reason)
   - Request additional information
   - Add review notes
5. **Review Submission** ‚Üí Admin:
   - Fills out review notes
   - Selects decision: Approve/Reject/Request Info
   - Clicks "Submit Review"
6. **Confirmation** ‚Üí Review saved, user notified

**Success Criteria**: Admin successfully reviews and processes submission

**Error Paths**:
- **Error**: Submission already reviewed by another admin
  - **Recovery**: Show current status, prevent duplicate review
- **Error**: Required review notes missing
  - **Recovery**: Highlight required fields, disable submit

**Pages Involved**: Admin Dashboard, Submissions List Page, Submission Detail Page, Review Page

---

### Flow 10: Manage Form Schema

**Trigger**: Admin wants to update form schema

**Steps**:
1. **Admin Dashboard** ‚Üí Admin clicks "Manage Forms"
2. **Forms List** ‚Üí Admin sees all form schemas:
   - Form name, version, status (active/inactive)
   - Last updated date
   - Number of submissions
3. **Form Schema Editor** ‚Üí Admin clicks form to edit:
   - View form schema JSON
   - Edit form fields: Add/remove/modify fields
   - Preview form: See rendered form preview
4. **Schema Validation** ‚Üí System validates schema changes
5. **Save Schema** ‚Üí Admin saves changes:
   - Version increment
   - Change log entry
6. **Form Preview** ‚Üí Admin previews updated form
7. **Activate Form** ‚Üí Admin activates new version

**Success Criteria**: Admin successfully updates form schema

**Error Paths**:
- **Error**: Schema validation fails
  - **Recovery**: Highlight errors, show validation messages
- **Error**: Breaking changes detected
  - **Recovery**: Warning message, require confirmation

**Pages Involved**: Admin Dashboard, Forms List Page, Form Schema Editor, Form Preview Page

---

### Flow 11: Export Submission Data

**Trigger**: Admin wants to export submission data for analysis

**Steps**:
1. **Admin Dashboard** ‚Üí Admin clicks "Reports & Analytics"
2. **Export Page** ‚Üí Admin configures export:
   - Select forms: All forms or specific forms
   - Filter by: Status, Date range, Form type
   - Format: CSV, Excel, PDF
   - Include: Form data, Documents, Audit trail
3. **Generate Export** ‚Üí Admin clicks "Generate Export"
4. **Export Progress** ‚Üí System generates export file
5. **Download Export** ‚Üí Admin downloads export file

**Success Criteria**: Admin successfully exports submission data

**Error Paths**:
- **Error**: Export too large
  - **Recovery**: Split into multiple files, or reduce date range
- **Error**: Export generation fails
  - **Recovery**: Error message, retry option

**Pages Involved**: Admin Dashboard, Export Page

---

### Flow 12: View Audit Logs

**Trigger**: Admin wants to review system audit logs

**Steps**:
1. **Admin Dashboard** ‚Üí Admin clicks "Audit Logs"
2. **Audit Logs Page** ‚Üí Admin sees filtered logs:
   - Filter by: Action type, User, Date range
   - Sort by: Date, Action type
   - Search: By user name, action, entity
3. **Log Detail** ‚Üí Admin clicks log entry to view:
   - Action: Create, Update, Delete, Review
   - User: Who performed action
   - Timestamp: When action occurred
   - Details: Before/after values, IP address
4. **Export Logs** ‚Üí Admin can export logs for compliance

**Success Criteria**: Admin successfully reviews audit logs

**Error Paths**:
- **Error**: Logs load slowly
  - **Recovery**: Pagination, lazy loading, loading indicator

**Pages Involved**: Admin Dashboard, Audit Logs Page
